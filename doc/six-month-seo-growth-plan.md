# PicZip 6-Month SEO Growth Plan

Goal: reach `10,000 UV/month` within 6 months.

Primary market: Google search, English first.

Core positioning: free browser image compressor, no sign-up, no upload, local processing, exact KB targets, batch ZIP download.

## North Star

The goal is not just traffic. The goal is search traffic from people who need to compress images now.

Main conversion action:
- Upload or drop image into the compressor.

Secondary actions:
- Visit task pages such as `/compress-image-to-100kb`.
- Download compressed output.
- Return for repeated compression tasks.

## Growth Assumptions

- Month 1 is mostly crawl, index, and early long-tail impressions.
- Month 2 should start showing Search Console query data.
- Months 3-4 are where content and landing page iteration matter most.
- Months 5-6 should focus on pages that already show impressions but weak CTR or ranking.

Expected monthly UV path:

| Month | Target UV/month | Main Focus |
|---|---:|---|
| 1 | 300-800 | Indexing, technical SEO, first content |
| 2 | 1,000-2,000 | Long-tail pages, guide content, UX fixes |
| 3 | 2,500-4,000 | Expand content clusters, improve CTR |
| 4 | 4,500-6,500 | Data-driven page updates, backlinks |
| 5 | 7,000-8,500 | Scale winning topics and internal links |
| 6 | 10,000+ | Refresh, optimize, and compound winners |

## Weekly Operating Rhythm

Every week should produce visible assets, not just analysis.

Monday:
- Check Google Search Console.
- Pick 3 pages to improve or create.
- Decide one technical/UX improvement if needed.

Tuesday-Wednesday:
- Write or improve content.
- Update title/meta/H1/FAQ/internal links.
- Add useful screenshots or examples if relevant.

Thursday:
- Publish changes.
- Submit important changed URLs in Search Console.
- Check Vercel deployment and live page output.

Friday:
- Review Search Console query movement.
- Record what changed.
- Pick next week's backlog.

## Daily Task Template

When asked "今天需要做什么", choose tasks in this order:

1. If Search Console is not set up: complete Search Console setup.
2. If sitemap is not submitted: submit sitemap.
3. If fewer than 10 pages are indexed: inspect and request indexing for core pages.
4. If no new content was published this week: write or publish one guide.
5. If Search Console has impressions: improve one page with high impressions and low CTR.
6. If Core Web Vitals or mobile UX is weak: fix UX/performance.
7. If all above are handled: build backlinks or submit to relevant directories.

- Group them by intent:

- Create supporting articles for each group.

Minimum output:

Tasks:

  - JPG vs PNG vs WebP
  - browser image compressor vs upload tools

- Add more internal links to pages with positions 8-30.

Minimum output:

## Month 5 Plan

## Key Metrics

Track weekly:
- Indexed pages
- Organic clicks
- Organic impressions
- CTR
- Average position
- Top queries
- Top landing pages
- Compression starts if analytics is added
- Downloads if analytics is added

Warning signs:
- Pages indexed but no impressions after 4-6 weeks.
- Impressions growing but CTR below 1%.
- Users visit but do not use the compressor.
- High mobile bounce due to poor layout or slow loading.

## Rules For Future Decisions

- Do not add unrelated tools until compression pages show traction.
- Do not publish generic AI content.
- Do not buy low-quality backlinks.
- Do not hide the tool below marketing copy.
- Every new content page must link to the matching compression tool.
- Every tool landing page should answer a real search intent.
